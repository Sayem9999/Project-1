# Tracking Workflow

Use these files for every task:

- `docs/tracking/WORK_ITEMS.md`: entry point and task status
- `docs/tracking/BUG_REGISTER.md`: defects and root causes
- `docs/tracking/CHANGE_LOG.md`: implementation changes and rationale
- `docs/tracking/TEST_EVIDENCE.md`: test commands and outcomes

ID format:

- `WRK-YYYYMMDD-NNN`
- `BUG-YYYYMMDD-NNN`
- `CHG-YYYYMMDD-NNN`
- `TST-YYYYMMDD-NNN`
